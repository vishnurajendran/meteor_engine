This does not guarantee that all deferred loads will succeed, since an asset could be dependent on another deferred load asset, at this point the order will dictate the asset-load success

◆ hasDeferredLoad()

virtual bool MAsset::hasDeferredLoad ( ) const
inlineoverridevirtualinherited

Inform that this asset needs deferred loading.

By default, the assets do not need deferred loading, but some assets like material will need to depend on another asset like shader to get all available properties for a specific shader. so setting this true, will allow the asset to have another pass at loading after all independent assets are loaded, giving the dependencies to be resolved without any issue

Implements IDefferedLoadableAsset.

Reimplemented in MCubemapAsset, and MMaterialAsset.
